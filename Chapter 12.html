Computer Arithmetic

Adding and Subtracting Positive Integers

Adding : To add two binary numbers we use a similar process to when we add numbers in binary, we align the number columns correctly and then go column by column from right to left adding them up. So 1+0=0 and if we get 1+1=2 which we cant do in binary, its like getting 9+2=11 in denary so we carry the excess over to the next column so we get 1+1=0 carry 1. If we carry over a 1 into a column that isn't there yet as we're limited to a certain number of bits then that means the number is too big to store in that many bits and would need more bits to be stored. This is called an overflow error.

Subtracting : To take two numbers away from each other we again use a similar process to denary, we subtract the columns so 1-1=0, 1-0=1 but then 0-1=-1? when this happens like in denary we 'borrow' 1 from the next column. As 10 in binary is 2 we then add 2 to the column we're borrowing into, this may lead to you having to borrow from multiple columns as there are fewer amounts you can borrow from a column than in denary.

Decimal Numbers In Binary

There are two main methods of storing decimal binary numbers, the first is fixed point and the second floating point. In fixed point the decimal point goes inbetween the 1 and the 1/2 in the same way as in denary it goes between 1 and 1/10, the issue with this is that you can only be accurate to whatever the last fraction column heading is so with 2bits after the decimal place you ca only be accurate to the nearest 1/4(0.25) which would be rubbish for storing money values etc.

Floating point is a little more difficult, for floating point numbers you first have to split the number is to two parts, the Mantissa and the Exponent. The mantissa represents the values of the number eg 62153 then the exponent represents the magnitude of the number, eg x10^-3 so combined the two examples leave 62.153. In binary you take the mantissa and use sign and magnitude method to format the number and move the number to the right of the decimal so 159.76 would be a mantissa of 0.15976, then you use the exponent to change the magnitude to the size you want, so to turn 0.15976 to 159.76 we need an exponent of 3 as the decimal moves 3 places to the right.

Adding and Subtracting In Two's Compliment

Adding : Adding two two's compliment numbers is the same process as in normal binary however when the number overflows the output results as a negative number so you'd need an extra bit to store numbers larger than 127.

Subtracting : For subtraction first use one's compliment (all 1's become 0's and vice versa then add 1) on the number that will be subtracted, then simply add the two numberrs.

Normalisation

For positive sign and magnitude numbers take your number and move the binary point over so that it is situated in front of the frist 1, then simply change the exponent accordingly. So if you moved the binary point three places to the right and the exponenent was originally 5 then minus three and make the exponent 2. For negative sign and magnitude move the binary point to the first 0 then save as with positive, adjust the exponent to match.
